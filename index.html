Unit I 1-Mark Questions

1. What is 8051?
8-bit microcontroller by Intel (1981), 40-pin DIP, Harvard architecture, 4KB ROM, 128B RAM.
2. How many I/O ports?
4 ports (P0-P3), 8-bit each = 32 I/O lines.
3. ROM/RAM sizes?
ROM: 4KB (0000H-0FFFH). RAM: 128 bytes (00H-7FH).
4. How many timers?
2 x 16-bit timers: Timer 0 and Timer 1.
5. How many interrupts?
6 sources: Reset + 5 vectored (INT0, T0, INT1, T1, Serial).
6. Name 5 addressing modes.
Immediate, Register, Direct, Register Indirect, Indexed.
7. What is PSW?
Program Status Word - 8-bit flag register (CY, AC, F0, RS1, RS0, OV, P).
8. DPTR size and use?
16-bit Data Pointer - used to access external memory.
9. What is SBUF?
Serial Buffer register - holds serial TX/RX data.
10. Function of ALE?
Address Latch Enable - demultiplexes address/data on Port 0.
11. What does EA pin do?
External Access - VCC=internal ROM, GND=external ROM only.
12. What is PSEN?
Program Store Enable - active-low read strobe for external program memory.
13. How many register banks?
4 banks (Bank 0-3), 8 registers each (R0-R7).
14. Which bits select register banks?
RS1 (PSW.4) and RS0 (PSW.3).
15. 4 timer modes?
Mode 0: 13-bit, Mode 1: 16-bit, Mode 2: 8-bit auto-reload, Mode 3: split.
16. Timer vs Counter?
Timer counts machine cycles. Counter counts external pulses.
17. What is SCON?
Serial Control register - selects mode, enables receiver, TI/RI flags.
18. Interrupt vector addresses?
Reset-0000H, INT0-0003H, T0-000BH, INT1-0013H, T1-001BH, Serial-0023H.
19. What is IE register?
Interrupt Enable - EA (global), ES, ET1, EX1, ET0, EX0.
20. Data/Address bus sizes?
8-bit data bus, 16-bit address bus (64KB addressable).

Unit I 16-Mark Questions

1 Explain 8051 architecture with block diagram. (16)

CPU: 8-bit processor with ALU for arithmetic/logical ops. Can process single-bit data (bit-addressable).

ROM (4KB): Program memory, address 0000H-0FFFH. External expansion up to 64KB.

RAM (128B): Register banks (00H-1FH, 4x8), Bit-addressable (20H-2FH, 128 bits), Scratchpad (30H-7FH). SFRs at 80H-FFH.

I/O Ports: 4 parallel ports (P0-P3), 8-bit each = 32 I/O lines. Bidirectional. P0 also for address/data.

Timers: Two 16-bit timer/counters (T0, T1) for delay, counting, baud rate.

Serial Port: Full-duplex UART. SBUF, SCON registers. 4 modes.

Interrupts: 5 vectored sources + Reset. 2 priority levels. IE, IP, TCON registers.

Buses: 8-bit data, 16-bit address. Control: PSEN, ALE, EA, RST. Harvard architecture means separate program/data memory buses for simultaneous access.

2 Explain 8051 pin diagram with all 40 pins. (16)

Power: VCC (40)=+5V, GND (20)=0V.

Oscillator: XTAL1 (19) input, XTAL2 (18) output. 4-30 MHz crystal.

Control: RST (9) reset active-high, PSEN (29) ext ROM strobe, ALE (30) address latch, EA (31) ext access.

Port 0 (32-39): Bidirectional I/O, no internal pull-ups. Multiplexed AD0-AD7.

Port 1 (1-8): Bidirectional I/O with internal pull-ups.

Port 2 (21-28): Bidirectional I/O. A8-A15 for external memory.

Port 3 (10-17): I/O + alt functions: RXD, TXD, INT0, INT1, T0, T1, WR, RD.

3 Explain all 5 addressing modes with examples. (16)

1. Immediate: MOV A, #6AH - Data directly in instruction. '#' prefix indicates immediate.

2. Direct: MOV A, 04H - Address of operand given directly. No '#' prefix.

3. Register Direct: MOV A, R4 - Register name as operand. R0-R7 from current bank. 1-byte instruction.

4. Register Indirect: MOV A, @R0 - Register holds address of data. Only R0,R1. '@' indicates indirection.

5. Indexed: MOVC A, @A+DPTR - Access program memory. Address = A + DPTR or A + PC. For lookup tables.

4 Explain 8051 timer/counter modes and serial communication. (16)

TMOD Register: GATE, C/T (timer/counter), M1, M0 bits for each timer.

4 Modes: Mode 0 (13-bit), Mode 1 (16-bit, most common), Mode 2 (8-bit auto-reload for baud rate), Mode 3 (split timer, T0 only).

Delay Calculation: For 12MHz, machine cycle = 1us. For 1ms delay in Mode 1: Load = 65536 - 1000 = FC18H.

Serial (UART): SCON selects mode. SBUF holds data. TI/RI flags. 4 modes: 0 (shift register), 1 (8-bit UART), 2 (9-bit fixed baud), 3 (9-bit variable baud). Baud rate from Timer 1.

5 Explain 8051 interrupt system with registers and priority. (16)

6 Interrupt Sources: Reset (0000H), INT0 (0003H), T0 (000BH), INT1 (0013H), T1 (001BH), Serial TI/RI (0023H).

IE Register: EA (global enable), ES (serial), ET1, EX1, ET0, EX0. 1 = enable.

IP Register: PT1, PX1, PT0, PX0. 1 = high priority, 0 = low priority.

TCON: IT0/IT1 (edge=1/level=0), IE0/IE1, TF0/TF1 flags.

Process: Flag set -> Check enables -> Save PC to stack -> Jump to vector -> ISR runs -> RETI restores PC.

Embedded C Programming

Memory Interfacing - Embedded C - RTOS - Context Switching - Scheduling

Unit II Key Points

Microprocessor vs Microcontroller
Micro = CPU only. Microcontroller = CPU + memory + I/O on single chip.
What is RTOS?
Real-Time OS for deterministic task scheduling with timing guarantees.
Types of RTOS
Hard (strict), Soft (relaxed), Firm (quality impact).
Context Switching
Save current task state, restore another to resume execution.
What is PCB?
Process Control Block - stores PC, registers, stack pointer, memory info.
Preemptive vs Non-Preemptive
Preemptive: can interrupt. Non-Preemptive: runs to completion.
'volatile' keyword
Prevents compiler optimization - variable changes by hardware.
'sbit' keyword
Declares single bit in SFR. Example: sbit LED = P1^0;
GPOS vs RTOS
GPOS: fairness, avg bandwidth, non-deterministic. RTOS: guaranteed response, strict timing.
What is a semaphore?
Synchronization primitive for shared resource access control.
Priority Inversion
Lower priority task holds resource needed by higher priority task.
Round Robin
Each task gets fixed time slice in circular order.

Unit II 16-Mark Questions

1 Explain external memory interfacing in 8051. (16)

Program Memory: P0 multiplexes A0-A7/D0-D7 with ALE latching address in 74HC373. P2 provides A8-A15. PSEN enables external ROM. EA=VCC for internal+external. Max 64KB.

Data Memory: Same address/data. RD (P3.7) and WR (P3.6) control RAM. MOVX instruction accesses external RAM. Max 64KB.

Internal Memory: RAM: Register Banks (00H-1FH), Bit-addressable (20H-2FH), Scratchpad (30H-7FH). SFRs (80H-FFH) for control registers.

2 Explain Embedded C concepts with code examples. (16)

sfr: Declares SFR at specific address. sfr P0 = 0x80;

sbit: Declares single bit of SFR. sbit LED = P1^0;

volatile: Prevents optimization for hardware-changing variables.

Bit manipulation: SET_BIT(reg,bit) = (reg |= (1<

Example - LED Blink: #include <reg51.h> sbit LED = P1^0; void delay(unsigned int ms) { ... } void main() { while(1) { LED = 0; delay(1000); LED = 1; delay(1000); } }

3 Explain RTOS - need, types, components, scheduling. (16)

Need: Multiple tasks with timing requirements, deterministic response, modular development, efficient resource management.

Types: Hard RTOS (airbag, flight control), Soft RTOS (multimedia), Firm RTOS (quality impact but not fatal).

Components: Scheduler, Task Manager, Inter-task Communication (queues, semaphores, mutexes), Memory Manager, Timer/ISR handlers.

Scheduling: Preemptive (interrupt driven), Non-Preemptive (cooperative), Rate Monotonic (fixed priority by period), EDF (dynamic by deadline), Round Robin.

4 Explain context switching and priority scheduling. (16)

Context Switching: Saving/restoring task state (PC, registers, SP, memory info) in PCB. Triggered by interrupts, timer ticks, system calls.

Flow: Task A Running -> Interrupt -> Save A's PCB -> Scheduler selects B -> Restore B's PCB -> Task B Running.

Preemptive: Higher priority interrupts lower. Common in RTOS. Emergency brake interrupts cruise control.

Non-Preemptive: Task runs until it yields. Simple but starvation risk.

RM: Shorter period = higher priority. EDF: Nearest deadline = highest priority (up to 100% utilization).

Unit IV 16-Mark Questions

1 Compare Bluetooth, WiFi, ZigBee, GPS, and GSM. (16)

Bluetooth (IEEE 802.15.1): 2.4 GHz, 10m, 1-3 Mbps, low power. Piconet/scatternet. BLE for IoT. Use: headsets, sensor data.

WiFi (IEEE 802.11): 2.4/5 GHz, 100m, 54-600 Mbps, high power. Star topology with AP. Use: internet, streaming, home networks.

ZigBee (IEEE 802.15.4): 2.4 GHz, 100m mesh, 250 Kbps, very low power. Self-healing mesh. Use: home automation, smart lighting, industrial monitoring.

GPS: 1.5 GHz L1, global, no uplink, 3-15m accuracy. 24+ satellites. Use: navigation, fleet tracking, geofencing.

GSM: 900/1800 MHz, cellular, up to 35km. 2G-4G. SIM auth. AT commands (SIM800). Use: SMS alerts, voice, NB-IoT.

Selection Criteria: Range required, data rate, power budget, cost per node, topology (star/mesh), interoperability.

2 Explain IoT communication models and cloud connectivity. (16)

4 Models:

1. Request-Response: Client sends HTTP request, server responds. REST APIs.

2. Publish-Subscribe: Devices publish to topics. Subscribers receive via broker. MQTT is most common. Decoupled architecture.

3. Push-Pull: Producers push to queue. Consumers pull. Buffering and load-leveling.

4. Exclusive Pair: Direct persistent bidirectional connection. Real-time control.

Cloud Connectivity Flow: Device collects sensor data -> Connects via WiFi/GSM -> Sends via HTTP/MQTT -> Cloud stores/processes -> User accesses via app.

MQTT Example: ESP32 reads temp -> publishes "home/temp" -> Raspberry Pi broker receives -> Node-RED dashboard -> if temp > threshold -> publish "home/ac/on".

8051 Architecture Deep Dive

Think of 8051 as a factory:
CPU = Factory Manager
ROM = Blueprint Cabinet
RAM = Workbench
I/O Ports = Loading Docks
Timers = Alarm Clocks
Interrupts = Doorbells

What is Harvard Architecture?

8051 uses Harvard architecture - separate buses for program (ROM) and data (RAM). Can read instruction AND access data simultaneously - faster than Von Neumann.

Why 8051 is still popular?

  • Simple & easy to learn - perfect for beginners
  • Widely available and cheap
  • Huge community support
  • Bit-addressable - work with individual bits
  • Used in millions of products

8051 Instruction Set & Programming

; 8051 Assembly - Add two numbers ORG 0000H MOV A, #25H ; Load 25H into A MOV R1, #14H ; Load 14H into R1 ADD A, R1 ; A = 25H + 14H = 39H MOV P1, A ; Send result to Port 1 END

Mnemonics: MOV=move, ADD=add, SUBB=subtract with borrow, JMP=jump, CJNE=compare+if not equal jump, DJNZ=decrement+if not zero jump.

Timers & Serial Port

Timer 0 & Timer 1: Both 16-bit. THx (high byte) + TLx (low byte).

4 Timer Modes:

  • Mode 0: 13-bit timer (THx 8-bit + TLx 5-bit). Max count = 8192.
  • Mode 1: 16-bit timer. Max count = 65536. Most common.
  • Mode 2: 8-bit auto-reload. TLx counts, THx holds reload value. Used for baud rate.
  • Mode 3: Split timer (T0 only). TL0 as 8-bit timer, TH0 as 8-bit timer.

Delay Calculation (12MHz):

Machine cycle = 12/12MHz = 1us. For 1ms: Load = 65536 - 1000 = 64536 (FC18H).

Serial Communication:

SCON selects mode. SBUF holds data. TI flag = transmit complete. RI flag = receive complete. Baud rate from Timer 1: Baud = (2^SMOD/32) x (Fosc/(12x(256-TH1))).

Interrupt Handling in 8051

6 Interrupt Sources: Reset (0000H), INT0 (0003H), Timer 0 (000BH), INT1 (0013H), Timer 1 (001BH), Serial (0023H).

IE Register: EA (global), ES (serial), ET1 (T1), EX1 (INT1), ET0 (T0), EX0 (INT0).

IP Register: PT1, PX1, PT0, PX0 - set = high priority.

Process: Flag set -> Check enables -> Save PC to stack -> Vector address -> ISR -> RETI restores PC.

Analogy: "Interrupt is like a doorbell. You're cooking (main program), doorbell rings, you answer (ISR), then resume cooking exactly where you left off."
