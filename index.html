🔄 Bare Metal vs RTOS

🪨

Bare Metal

Single infinite loop. No OS. One task at a time. Simple but can't handle multiple things simultaneously.

⚡

RTOS

Multiple tasks. Scheduler decides which runs. Context switching enables "multitasking" on single CPU. Essential for time-critical IoT.

🔄 Context Switching Flow

Task A Running → Interrupt! → Save A's State (PCB) → Load Task B's State → Task B Running
💡 "Context switching is like bookmarking a page in a book, then switching to another book. When you come back, you resume from your bookmark."

🏗️ 8051 Architecture — Deep Dive

💡 Think of 8051 as a factory:
CPU = Factory Manager (makes decisions)
ROM = Blueprint Cabinet (permanent plans)
RAM = Workbench (temporary workspace)
I/O Ports = Loading Docks (talk to outside)
Timers = Alarm Clocks (measure time)
Interrupts = Doorbells (urgent messages)

What is Harvard Architecture?

8051 uses Harvard architecture — separate memory for program (ROM) and data (RAM) with separate buses. This means it can read an instruction AND access data at the same time, making it faster than Von Neumann architecture.

Why 8051 is still popular?

  • Simple & easy to learn — perfect for beginners
  • Widely available and cheap
  • Huge community support and resources
  • Bit-addressable — can work with individual bits
  • Used in millions of products (toys, appliances, automotive)

💻 8051 Instruction Set & Programming

; 8051 Assembly — Add two numbers
ORG 0000H       ; Start at address 0
MOV A, #25H     ; Load 25 into accumulator
MOV R1, #14H    ; Load 14 into register R1
ADD A, R1       ; Add R1 to A (A = 39H)
MOV P1, A       ; Send result to Port 1
END             ; End of program

5 Addressing Modes

ModeExampleMeaning
ImmediateMOV A, #25HLoad value directly
RegisterADD A, R1Use register value
DirectMOV A, 40HUse RAM address
IndirectMOV A, @R0Use address in register
IndexedMOVC A, @A+DPTRProgram memory access
💡 Mnemonics are shortcuts: MOV = move, ADD = add, SUBB = subtract with borrow, JMP = jump, CJNE = compare and jump if not equal, DJNZ = decrement and jump if not zero.

What are Timers used for?

  • Delay generation — create precise time delays (e.g., blinking LED every 1 second)
  • Baud rate control — set speed for serial communication
  • Event counting — count external pulses (e.g., count products on conveyor belt)
  • PWM generation — create pulse width modulated signals
💡 "Think of timers as stopwatches. You set a value, the timer counts down/up, and when it reaches zero, it sets a flag or triggers an interrupt."

⚡ Interrupt Handling in 8051

5 Interrupt Sources

Reset (0000H)
INT0 (0003H)
Timer 0 (000BH)
INT1 (0013H)
Timer 1 (001BH)
Normal Program ← interrupt! → ISR Runs → Resume Program
💡 "An interrupt is like a doorbell. You're cooking (main program), doorbell rings (interrupt), you stop cooking, answer the door (ISR), then go back to cooking exactly where you left off."

Priority Levels

Each interrupt can be set to high or low priority using the IP (Interrupt Priority) register. If two interrupts occur simultaneously, the higher priority one runs first. Reset has the highest priority.

⌨️ Embedded C Programming

// Blink LED using embedded C
#include <reg51.h>   // 8051 register definitions

sbit LED = P1^0;     // LED connected to pin P1.0

void delay(unsigned int ms) {
    unsigned int i, j;
    for(i = 0; i < ms; i++)
        for(j = 0; j < 127; j++);  // ~1ms delay
}

void main() {
    while(1) {
        LED = 1;      // Turn ON
        delay(1000);   // Wait 1 second
        LED = 0;      // Turn OFF
        delay(1000);   // Wait 1 second
    }
}

Key Embedded C Concepts

  • volatile — tells compiler not to optimize variable (may change by hardware)
  • sbit — access single bit of SFR (e.g., sbit LED = P1^0)
  • sfr — declare special function register at specific address
  • bit manipulation — use &, |, ^, ~ to set/clear/toggle individual bits
  • delay loops — nested loops for approximate timing (not precise)

🔄 Why RTOS is Needed?

Without RTOS vs With RTOS

❌ Without RTOS

One long loop — tasks run sequentially. Critical tasks wait!

✅ With RTOS

Tasks share CPU time. High-priority tasks run first.

💡 "RTOS is like a restaurant manager. Without RTOS, the chef does everything alone (cook, serve, clean). With RTOS, the manager assigns tasks: chef cooks, waiter serves, cleaner cleans — all working together."

🔄 Context Switching — Explained Simply

Task A → Save → Task B

Animation: Task A running → Interrupt → Save state → Task B runs

What is saved in PCB?

  • Program Counter (PC) — where to resume
  • CPU registers (A, B, R0-R7, DPTR, PSW)
  • Stack pointer
  • Memory allocation info
  • I/O status
💡 "Context switching is like pausing a movie, switching to another channel, then coming back to the exact frame you paused."

📊 Priority Scheduling

Preemptive vs Non-Preemptive

⏭️ Preemptive

Higher priority task can interrupt a running lower priority task. CPU saves state, runs high priority, resumes.

⏸️ Non-Preemptive

Task runs until it voluntarily releases CPU (completes or blocks). No interruptions once started.

💡 "Preemptive = your friend interrupts your conversation for an urgent call. Non-preemptive = your friend waits until you finish talking."

void setup() {
  // Runs ONCE at power-on
  // Configure pins, start serial
  pinMode(LED_BUILTIN, OUTPUT);

}

void loop() {
  // Runs FOREVER
  // Main program logic

  delay(1000);
  digitalWrite(LED_BUILTIN, LOW);
  delay(1000);
}
