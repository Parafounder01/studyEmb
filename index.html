Unit I 1-Mark Questions (20)

1. What is 8051?
8-bit microcontroller by Intel (1981), 40-pin DIP, Harvard architecture, 4KB ROM, 128B RAM.
2. How many I/O ports?
4 ports (P0-P3), 8-bit each = 32 I/O lines.
3. ROM/RAM sizes?
ROM: 4KB (0000H-0FFFH). RAM: 128 bytes (00H-7FH).
4. How many timers?
2 x 16-bit timers: Timer 0 and Timer 1.
5. How many interrupt sources?
5 vectored (INT0, T0, INT1, T1, Serial) + Reset = 6 total.
6. Name 5 addressing modes.
Immediate, Register, Direct, Register Indirect, Indexed.
7. What is PSW?
Program Status Word - 8-bit flag register (CY, AC, F0, RS1, RS0, OV, P).
8. DPTR size and use?
16-bit Data Pointer — used to access external memory (MOVX, MOVC).
9. What is SBUF?
Serial Buffer register — holds TX/RX data for serial port.
10. Function of ALE?
Address Latch Enable — demultiplexes address/data on Port 0.
11. What does EA pin do?
External Access — VCC=internal+external ROM, GND=external only.
12. What is PSEN?
Program Store Enable — active-low read strobe for external program memory.
13. How many register banks?
4 banks (Bank 0-3), 8 registers each (R0-R7).
14. Which bits select register banks?
RS1 (PSW.4) and RS0 (PSW.3). 00=Bank0, 01=Bank1, 10=Bank2, 11=Bank3.
15. 4 timer modes?
Mode0: 13-bit, Mode1: 16-bit, Mode2: 8-bit auto-reload, Mode3: split timer.
16. Timer vs Counter?
Timer counts machine cycles (internal). Counter counts external pulses on T0/T1 pins.
17. What is SCON?
Serial Control register — selects mode, enables receiver, TI/RI flags.
18. Interrupt vector addresses?
Reset-0000H, INT0-0003H, T0-000BH, INT1-0013H, T1-001BH, Serial-0023H.
19. What is IE register?
Interrupt Enable — EA (global), ES, ET1, EX1, ET0, EX0 bits.
20. What is the initial SP value?
SP = 07H after reset. Stack grows upward from 08H in internal RAM.

Unit I 16-Mark Questions

1 Explain 8051 architecture with block diagram. (16)

CPU: 8-bit processor with ALU for arithmetic/logic ops. Can process single-bit data (bit-addressable).

ROM (4KB): Program memory, address 0000H-0FFFH. External expansion up to 64KB via P0/P2.

RAM (128B): Register banks (00H-1FH, 4x8=32B), Bit-addressable (20H-2FH, 128 bits), Scratchpad (30H-7FH). SFRs at 80H-FFH.

I/O Ports: 4 parallel ports (P0-P3), 8-bit each = 32 I/O lines. Bidirectional. P0 also for AD0-AD7. P2 for A8-A15. P3 has alternate functions.

Timers: Two 16-bit timer/counters (T0, T1). TMOD config. TCON flags. 4 modes.

Serial Port: Full-duplex UART. SBUF, SCON registers. 4 modes. Baud from Timer 1.

Interrupts: 5 vectored + Reset. IE/IP registers. 2 priority levels.

Buses: 8-bit data, 16-bit address. Control: PSEN, ALE, EA, RST. Harvard architecture with separate buses for program/data.

2 Explain 8051 pin diagram with all 40 pins. (16)

Power: VCC (40)=+5V, GND (20)=0V.

Oscillator: XTAL1 (19)=osc input, XTAL2 (18)=osc output. 4-30 MHz crystal.

Control: RST (9)=reset active-high, PSEN (29)=ext program strobe, ALE (30)=address latch enable, EA (31)=external access.

Port 0 (32-39): Bidirectional I/O, no internal pull-ups (open-drain). Multiplexed AD0-AD7 for external memory.

Port 1 (1-8): Bidirectional I/O with internal pull-ups.

Port 2 (21-28): Bidirectional I/O. Provides A8-A15 for external memory addressing.

Port 3 (10-17): I/O + alternate functions: RXD(10), TXD(11), INT0(12), INT1(13), T0(14), T1(15), WR(16), RD(17).

3 Explain all 5 addressing modes with examples. (16)

1. Immediate: MOV A, #6AH — Data directly in instruction. 2-byte instruction.

2. Direct: MOV A, 04H — Address of operand given directly. No '#'. 2-3 bytes. Accesses RAM 00H-7FH and SFRs 80H-FFH.

3. Register Direct: MOV A, R4 — Register name as operand. R0-R7 from current bank. 1-byte instruction.

4. Register Indirect: MOV A, @R0 — Register holds address of data. Only R0,R1 allowed. 1-byte.

5. Indexed: MOVC A, @A+DPTR — Access program memory for lookup tables. Read-only.

4 Explain 8051 timer/counter modes and serial communication. (16)

TMOD Register: GATE, C/T (timer/counter select), M1, M0 bits for each timer.

4 Modes: Mode 0 (13-bit, max 8192), Mode 1 (16-bit, most common, max 65536), Mode 2 (8-bit auto-reload, used for baud rate), Mode 3 (split timer, T0 only).

Delay Calculation: For 12MHz, machine cycle = 1us. For 1ms in Mode 1: Load = 65536 - 1000 = FC18H.

Serial (UART): SCON selects mode. SBUF holds data. TI/RI flags. Baud = (2SMOD/32) x (Fosc/(12x(256-TH1))).

5 Explain 8051 interrupt system with registers and priority. (16)

6 Interrupt Sources: Reset (0000H, highest), INT0 (0003H), T0 (000BH), INT1 (0013H), T1 (001BH), Serial TI/RI (0023H).

IE Register (A8H): EA (7)=global, ES(4)=serial, ET1(3)=T1, EX1(2)=INT1, ET0(1)=T0, EX0(0)=INT0.

IP Register (B8H): PT1(3), PX1(2), PT0(1), PX0(0). 1=high priority, 0=low priority.

TCON (88H): IT0/IT1 (edge=1/level=0 trigger), IE0/IE1 (pending flags), TF0/TF1 (timer overflow flags).

Process: Flag set → Check IE enable bits → Save PC to stack → Jump to vector address → ISR runs → RETI restores PC from stack.

Unit II 1-Mark Questions (20)

1. What are 3 reasons for a design methodology?
Scorecard tracking, early bug detection, team coordination.
2. Major non-functional requirements?
Cost, power, performance, size, time-to-market.
3. What is WCET?
Worst-Case Execution Time — maximum time a task can take.
4. What is a conceptual specification?
User-facing description of purpose, inputs, outputs, functions.
5. Two parts of cost?
Manufacturing cost (per unit) + R&D cost (one-time).
6. What does the Model Train Controller do?
Controls speed/direction of up to 8 model trains via track UART.
7. ARM stands for?
Advanced RISC Machine — RISC architecture family.
8. ARM7TDMI pipeline stages?
Fetch, Decode, Execute — 3-stage pipeline.
9. What does T in ARM7TDMI mean?
Thumb — 16-bit compressed instruction set.
10. ARM has how many registers?
37 registers total: 30 general-purpose + 7 status registers.
11. What is CPSR?
Current Program Status Register — holds condition flags (NZCV) + mode bits + IRQ/FIQ disable.
12. How many ARM processor modes?
7 modes: User, FIQ, IRQ, Supervisor, Abort, Undefined, System.
13. What is SPSR?
Saved Program Status Register — saves CPSR when exception occurs.
14. Which instruction enters Supervisor mode?
SWI (Software Interrupt) with 24-bit immediate argument.
15. ARM addressing: base-plus-offset?
Register value + offset forms address. Eg: LDR R0, [R1, #4]
16. Compare CMP instruction?
CMP R0,R1 computes R0-R1, sets status bits, discards result.
17. What is LDR/STR?
LDR=Load Register (memory to reg). STR=Store Register (reg to memory). ARM is load-store architecture.
18. LDMIA means?
Load Multiple, Increment After — loads multiple registers in one instruction.
19. What does BL instruction do?
Branch with Link — calls subroutine, saves return address in LR (R14).
20. What is the barrel shifter?
Part of ARM datapath that shifts/rotates second operand before ALU operation — at no extra cycle cost.

Unit II 16-Mark Questions

1 Explain the embedded system design process with the model train controller example. (16)

Design Process Steps: 1) Requirements gathering, 2) Conceptual specification, 3) Detailed specification (CPU, memory, I/O), 4) Architecture design, 5) Component design, 6) System integration, 7) Testing.

Model Train Controller: Goal: control speed of up to 8 model trains. Inputs: speed/direction buttons. Outputs: track voltage. One-way communication (track to train). Protocol uses CRC for error detection. Based on NEM 680/681 physical layer.

Key Non-functional: Cost (manufacturing + development), Performance (WCET analysis), Power (wall-powered for controller), Size.

2 Explain ARM processor architecture (ARM7TDMI). (16)

ARM Family: ARM is a RISC architecture family. ARM does not manufacture — it licenses IP to partners. Versions: ARMv4, ARMv4T, ARMv5T, ARMv5TE, ARMv6.

ARM7TDMI: T=Thumb (16-bit), D=Debug, M=Multiplier (64-bit result), I=EmbeddedICE.

3-Stage Pipeline: Fetch (from memory), Decode (decode instruction), Execute (ALU/register access).

Registers: 37 total (30 general-purpose + 7 status). R0-R12 general, R13=SP, R14=LR, R15=PC. CPSR holds NZCV flags + mode bits (M[4:0]) + IRQ/FIQ disable bits.

7 Processor Modes: User, FIQ, IRQ, Supervisor, Abort, Undefined, System.

Features: Conditional execution on every instruction, barrel shifter for free shifts, load-store architecture, multiple register load/store (LDM/STM).

3 Explain ARM instruction set — data processing, branch, multiply, and load/store. (16)

Data Processing: ADD, SUB, RSB (reverse subtract), AND, ORR, EOR, BIC (bit clear). Format: ADD R0, R1, R2 (R0 = R1 + R2). Immediate: ADD R0, R1, #5.

Multiply: MUL R4, R5, R6 (R4 = R5*R6). MLA R4, R5, R6, R7 (R4 = R5*R6 + R7). Restrictions: operands must be different registers.

Branch: B (unconditional), BL (branch+link for subroutines), BX (branch+exchange to Thumb). B{cond} for conditional.

Load/Store: LDR (load word), STR (store word). LDRB/STRB for bytes. Pre-indexed LDR R0, [R1, #4]!. Post-indexed LDR R0, [R1], #4. LDM/STM for multiple registers.

CMP: CMP R0,R1 subtracts and sets flags without storing result.

4 Explain supervisor mode, exceptions, assembly linking & loading. (16)

Supervisor Mode: Privileged mode entered via SWI instruction or reset. Allows access to protected resources. SWI takes 24-bit argument passed to supervisor handler.

Exceptions: Reset (highest priority), Data Abort, FIQ, IRQ, Prefetch Abort, SWI, Undefined Instruction. Each saves PC to LR_mode and CPSR to SPSR_mode.

Assembly - Linking - Loading: Assembly (.s) → Assembler translates to object file (.o) with relocation tables. Linker resolves symbols across .o files, assigns absolute addresses. Loader copies linked binary into memory, sets up stack, starts execution.

Performance Analysis: WCET analysis — measure worst-case instruction count x CPI. Profiling tools identify bottlenecks.

RTOS

Real-Time Operating Systems

Deterministic scheduling with timing guarantees. Hard RTOS (airbag, flight control) vs Soft (multimedia) vs Firm. Microkernel design: scheduler, timers, interrupts, system calls.

Scheduling

Tasks, Scheduling & Priority

Process states: waiting, ready, executing. Scheduling policies: Rate Monotonic (RMS, static), Earliest Deadline First (EDF, dynamic), Round Robin. Utilization analysis.

IPC

Interprocess Communication

Shared memory, message passing, semaphores (binary/counting), mutexes, mailboxes, queues. Priority inversion & inheritance. Deadlock prevention.

MPSoC

Distributed Systems & MPSoCs

Multiprocessor SoCs for high performance. Shared memory vs message passing. Design examples: Audio Player, Engine Control Unit, Video Accelerator.

RTOS Microkernel Structure

Scheduler Task Manager IPC Timer ISR Syscalls
RTOS vs GPOS: GPOS = fairness, avg bandwidth, non-deterministic. RTOS = guaranteed response, strict timing, preemptive priority scheduling (FreeRTOS, VxWorks, uC/OS).

Scheduling Policies

RMS: Shorter period = higher priority. Utilization bound U = n(21/n-1) → ~69% for large n.

EDF: Nearest deadline = highest priority. Up to 100% utilization.

Round Robin: Fixed time slice per task.

Process States

Waiting: Can't execute (I/O, resource).
Ready: Can execute, waiting for CPU.
Executing: Running on CPU.
Context switch: Save PCB → scheduler → load PCB → resume.

IPC Mechanisms

Semaphore: Binary (0/1) or Counting (N). Synchronization primitive.
Mutex: Binary semaphore with priority inheritance.
Mailbox: Message queue with fixed-size messages.

Unit III 16-Mark Questions

1 Explain RTOS structure, scheduling policies (RMS & EDF), and timing analysis. (16)

RTOS Microkernel: Minimal kernel providing scheduler, timers, ISR handling, system calls. Three trigger sources: system calls, scheduling/timer tick, external interrupts.

Process States: Waiting, Ready, Executing. Scheduler selects from ready queue. Context switching: save PCB (PC, registers, SP) → choose next → restore PCB → run.

RMS (Rate-Monotonic): Static priority. Shorter period = higher priority. Utilization bound U = n(21/n - 1). For large n, limit = ln 2 ≈ 69.3%. If total utilization < bound, all deadlines met.

EDF (Earliest Deadline First): Dynamic — task with nearest deadline runs next. Up to 100% utilization possible.

Timing: Release time (when ready), deadline (worst allowed completion), period (for periodic tasks). Schedulability analysis validates all deadlines.

2 Explain IPC mechanisms, priority inversion, and design example (Audio Player). (16)

IPC Mechanisms: 1) Semaphores — binary (0/1) or counting (N). Wait() decrements, Signal() increments. 2) Mutex — binary semaphore with priority inheritance. 3) Mailbox/Queue — kernel-managed message passing.

Priority Inversion: Low-priority task L holds mutex. Medium-priority M preempts L. High-priority H waits for mutex, stuck behind M. Solution: Priority inheritance — L temporarily inherits H's priority.

Audio Player Design: Tasks: 1) Read from storage (periodic), 2) Decode audio (CPU-intensive), 3) Output to DAC (real-time). IPC: buffer queue between reader and decoder, double-buffered output. Scheduling: output task highest priority.

3 Explain distributed embedded systems, MPSoCs, and shared memory multiprocessors. (16)

Distributed Embedded Systems: Multiple processing nodes connected by network (CAN, Ethernet, I2C). Each node has own CPU + memory. Nodes communicate via message passing. Examples: automotive ECU network (CAN bus), home automation (ZigBee mesh).

MPSoCs: Multiple processor cores on single chip. Shared memory vs distributed memory. Homogeneous (all same) vs heterogeneous (CPU + DSP + GPU).

Shared Memory: All processors see same memory. Cache coherence protocols (MESI). Locking (spinlocks, mutexes) for synchronization.

ECU Example: Tasks — crankshaft position sensing (1ms), fuel injection (2ms), spark advance (1ms), lambda feedback (10ms), OBD diagnostics (100ms). RMS scheduling with hard deadlines. CAN bus for inter-ECU communication.

2 Explain IoT system management with NETCONF-YANG, platform methodology, and reference model. (16)

Unit I — 8-Bit Embedded Processor

1. 8051 architecture type?
Harvard architecture — separate program/data buses.
2. How many pins in 8051?
40 pins DIP. 32 I/O + 8 control/power.
3. Function of PSEN?
Program Store Enable — read strobe for external program memory.
4. Bit-addressable RAM area?
20H-2FH (16 bytes = 128 addressable bits).
5. TMOD register address?
89H. Configures Timer 0 (bits 0-3) and Timer 1 (bits 4-7).
6. What is SMOD bit?
PCON.7 — doubles serial baud rate when set.
7. What does RETI do?
Return from Interrupt — restores PC, re-enables interrupts.
8. External interrupt pins?
P3.2 (INT0) and P3.3 (INT1), active low.
9. Size of register bank?
8 bytes per bank (R0-R7). 4 banks = 32 bytes total.
10. Stack initial location?
SP = 07H after reset. Stack starts at 08H.
11. MOVX instruction use?
Access external data RAM via DPTR or Ri.
12. What is CJNE?
Compare and Jump if Not Equal — conditional branch.
13. DA instruction?
Decimal Adjust accumulator after BCD addition.
14. Max crystal frequency?
4-30 MHz typical. 12 MHz for standard baud rates.
15. PC size in 8051?
16-bit — addresses 64KB (0000H-FFFFH).
16. What is SJMP?
Short Jump — 2-byte, range -128 to +127 bytes.
17. Which SFRs are bit-addressable?
Addresses ending with 0H or 8H: P0(80H), TCON(88H), P1(90H), SCON(98H), P2(A0H), IE(A8H), P3(B0H), IP(B8H).
18. Machine cycles for MOV A,R4?
1 machine cycle. Register direct is fastest.
19. Role of A register?
Accumulator — default for arithmetic, data transfer, I/O.
20. SFR addresses range?
80H to FFH. Only 21 SFRs actually implemented.
1 8051 architecture detail. (16)
CPU (ALU+Control), 4KB ROM, 128B RAM (register banks + bit-addressable + scratchpad + SFRs), 4 I/O ports, 2 timers (4 modes), UART (4 modes), interrupt controller (6 sources, 2 priority levels). Harvard architecture, 64KB external address space.
2 Pin diagram of 8051. (16)
Power: VCC(40)=+5V, GND(20). Osc: XTAL1(19), XTAL2(18). Control: RST(9) reset, PSEN(29), ALE(30), EA(31). Ports: P0(32-39) AD0-AD7, P1(1-8), P2(21-28) A8-A15, P3(10-17) alt functions.
3 Addressing modes with examples. (16)
Immediate: MOV A,#6AH. Direct: MOV A,04H. Register: MOV A,R4. Indirect: MOV A,@R0. Indexed: MOVC A,@A+DPTR.
4 Timer/counter and serial communication. (16)
TMOD — GATE,C/T,M1,M0 per timer. 4 modes: 0(13-bit), 1(16-bit), 2(8-bit auto-reload), 3(split). Serial: SCON selects mode, SBUF data, TI/RI flags. Baud = (2^SMOD/32) x (Fosc/(12x(256-TH1))).
5 Interrupt system. (16)
6 sources: Reset(0000H), INT0(0003H), T0(000BH), INT1(0013H), T1(001BH), Serial(0023H). IE: EA+individual enables. IP: priority bits. TCON: ITx for edge/level, IEx pending, TFx overflow.

8051 Architecture Deep Dive

Think of 8051 as a factory:
CPU = Manager, ROM = Blueprint Cabinet, RAM = Workbench, I/O Ports = Docks, Timers = Alarm Clocks, Interrupts = Doorbells

Harvard Architecture

8051 uses Harvard architecture — separate buses for program (ROM) and data (RAM). Can read instruction AND access data simultaneously — faster than Von Neumann.

Key Components

  • ALU: 8-bit arithmetic + logic + single-bit operations
  • ACC: Accumulator, main work register
  • B Register: Used with ACC for MUL/DIV
  • PSW: Flags — CY, AC, F0, RS1, RS0, OV, P
  • DPTR: 16-bit data pointer
  • SP: Stack pointer (initial 07H)
  • PC: 16-bit program counter (64KB)

8051 Instruction Set & Programming

; 8051 Assembly — Add two numbers ORG 0000H MOV A, #25H ; Load 25H MOV R1, #14H ; Load 14H ADD A, R1 ; A = 25H + 14H = 39H MOV P1, A ; Result to Port 1 END

5 Addressing Modes

ModeExampleUse
ImmediateMOV A,#25HLoad value directly
RegisterADD A,R1Register value as operand
DirectMOV A,40HRAM address directly
IndirectMOV A,@R0Address in register
IndexedMOVC A,@A+DPTRProgram memory

Timers & Serial Port

Timer 0 & Timer 1: Both 16-bit. THx (high) + TLx (low). TMOD configures mode.

4 Timer Modes:

  • Mode 0: 13-bit (THx 8-bit + TLx 5-bit). Max = 8192.
  • Mode 1: 16-bit. Max = 65536. Most common.
  • Mode 2: 8-bit auto-reload. For baud rate generation.
  • Mode 3: Split timer (T0 only).

Delay @ 12MHz:

Machine cycle = 1us. For 1ms: Load = 65536 - 1000 = FC18H.

Serial:

SCON selects mode. SBUF data buffer. TI/RI flags. Baud = (2SMOD/32) x (Fosc/(12x(256-TH1))).

Interrupt Handling in 8051

6 Interrupt Sources: Reset(0000H), INT0(0003H), T0(000BH), INT1(0013H), T1(001BH), Serial(0023H).

IE Register (A8H): EA(7)=global, ES(4)=serial, ET1(3)=T1, EX1(2)=INT1, ET0(1)=T0, EX0(0)=INT0.

IP Register (B8H): PT1(3), PX1(2), PT0(1), PX0(0). 1=high priority.

TCON (88H): IT1/IT0 (1=edge, 0=level). IE1/IE0 (pending). TF1/TF0 (timer overflow).

Process: Flag set → Check IE → Save PC to stack → Vector → ISR → RETI restores PC.

Analogy: "Interrupt is like a doorbell. You're cooking, doorbell rings, you answer, then resume cooking."
